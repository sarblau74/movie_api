# Project Title
MYFLIX APPLICATION
Building the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

API Documentation

Endpoints

Movies

  • GET /movies - Retrieve all movies
  • GET /movies/:title - Retrieve a movie by title
  • GET /movies/genre/:genre - Retrieve movies by genre
  • GET /movies/director/:director - Retrieve movies by director
  • POST /movies - Add a new movie
  • PUT /movies/:title - Update a movie's details
  • DELETE /movies/:title - Delete a movie by title

Users

  • GET /users - Retrieve all users
  • GET /users/:username - Retrieve a user by username
  • POST /users - Add a new user
  • PUT /users/:username - Update a user's details
  • DELETE /users/:username - Delete a user by username

POST /users

Adds a new user to the system.

Request Body: JSON object with name and email fields.

POST /movies

Adds a new movie to the system.

Request Body: JSON object with title, director, genre, releaseDate, and rating fields.

DELETE /users/:userId

Deletes a user by ID.

Path Parameter: userId - ID of the user to delete.

DELETE /movies/:movieId

Deletes a movie by ID.

Path Parameter: movieId - ID of the movie to delete.

Notes

  • Ensure that you include all required fields in your POST requests.
  • For DELETE requests, ensure that you provide the correct ID.
